1081  /** Specialize a Func. This creates a special-case version of the
1082  * Func where the given condition is true. The most effective
1083  * conditions are those of the form param == value, and boolean
1084  * Params. Consider a simple example:
1085  \code
1086  f(x) = x + select(cond, 0, 1);
1087  f.compute_root();
1088  \endcode
1089  * This is equivalent to:
1090  \code
1091  for (int x = 0; x < width; x++) {
1092  f[x] = x + (cond ? 0 : 1);
1093  }
1094  \endcode
1095  * Adding the scheduling directive:
1096  \code
1097  f.specialize(cond)
1098  \endcode
1099  * makes it equivalent to:
1100  \code
1101  if (cond) {
1102  for (int x = 0; x < width; x++) {
1103  f[x] = x;
1104  }
1105  } else {
1106  for (int x = 0; x < width; x++) {
1107  f[x] = x + 1;
1108  }
1109  }
1110  \endcode
1111  * Note that the inner loops have been simplified. In the first
1112  * path Halide knows that cond is true, and in the second path
1113  * Halide knows that it is false.
1114  *
1115  * The specialized version gets its own schedule, which inherits
1116  * every directive made about the parent Func's schedule so far
1117  * except for its specializations. This method returns a handle to
1118  * the new schedule. If you wish to retrieve the specialized
1119  * sub-schedule again later, you can call this method with the
1120  * same condition. Consider the following example of scheduling
1121  * the specialized version:
1122  *
1123  \code
1124  f(x) = x;
1125  f.compute_root();
1126  f.specialize(width > 1).unroll(x, 2);
1127  \endcode
1128  * Assuming for simplicity that width is even, this is equivalent to:
1129  \code
1130  if (width > 1) {
1131  for (int x = 0; x < width/2; x++) {
1132  f[2*x] = 2*x;
1133  f[2*x + 1] = 2*x + 1;
