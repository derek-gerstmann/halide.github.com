22 /** Different ways to handle a tail case in a split when the
23  * factor does not provably divide the extent. */
24 enum class TailStrategy {
25  /** Round up the extent to be a multiple of the split
26  * factor. Not legal for RVars, as it would change the meaning
27  * of the algorithm. Pros: generates the simplest, fastest
28  * code. Cons: if used on a stage that reads from the input or
29  * writes to the output, constrains the input or output size
30  * to be a multiple of the split factor. */
31  RoundUp,
32 
33  /** Guard the inner loop with an if statement that prevents
34  * evaluation beyond the original extent. Always legal. The if
35  * statement is treated like a boundary condition, and
36  * factored out into a loop epilogue if possible. Pros: no
37  * redundant re-evaluation; does not constrain input our
38  * output sizes. Cons: increases code size due to separate
39  * tail-case handling; vectorization will scalarize in the tail
40  * case to handle the if statement. */
41  GuardWithIf,
42 
43  /** Prevent evaluation beyond the original extent by shifting
44  * the tail case inwards, re-evaluating some points near the
45  * end. Only legal for pure variables in pure definitions. If
46  * the inner loop is very simple, the tail case is treated
47  * like a boundary condition and factored out into an
48  * epilogue.
49  *
50  * This is a good trade-off between several factors. Like
51  * RoundUp, it supports vectorization well, because the inner
52  * loop is always a fixed size with no data-dependent
53  * branching. It increases code size slightly for inner loops
54  * due to the epilogue handling, but not for outer loops
55  * (e.g. loops over tiles). If used on a stage that reads from
56  * an input or writes to an output, this stategy only requires
57  * that the input/output extent be at least the split factor,
58  * instead of a multiple of the split factor as with RoundUp. */
59  ShiftInwards,
60 
61  /** For pure definitions use ShiftInwards. For pure vars in
62  * update definitions use RoundUp. For RVars in update
63  * definitions use GuardWithIf. */
64  Auto
65 };
