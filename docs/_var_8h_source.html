00039     /** Implicit var constructor. Implicit variables are injected
00040      * automatically into a function call if the number of arguments
00041      * to the function are fewer than its dimensionality and a
00042      * placeholder ("_") appears in its argument list. Defining a
00043      * function to equal an expression containing implicit variables
00044      * similarly appends those implicit variables, in the same order,
00045      * to the left-hand-side of the definition where the placeholder
00046      * ('_') appears.
00047      *
00048      * For example, consider the definition:
00049      *
00050      \code
00051      Func f, g;
00052      Var x, y;
00053      f(x, y) = 3;
00054      \endcode
00055      *
00056      * A call to f with the placeholder symbol \ref _
00057      * will have implicit arguments injected automatically, so f(2, \ref _)
00058      * is equivalent to f(2, \ref _0), where \ref _0 = Var::implicit(0), and f(\ref _)
00059      * (and indeed f when cast to an Expr) is equivalent to f(\ref _0, \ref _1).
00060      * The following definitions are all equivalent, differing only in the
00061      * variable names.
00062      *
00063      \code
00064      g(_) = f*3;
00065      g(_) = f(_)*3;
00066      g(x, _) = f(x, _)*3;
00067      g(x, y) = f(x, y)*3;
00068      \endcode
00069      *
00070      * These are expanded internally as follows:
00071      *
00072      \code
00073      g(_0, _1) = f(_0, _1)*3;
00074      g(_0, _1) = f(_0, _1)*3;
00075      g(x, _0) = f(x, _0)*3;
00076      g(x, y) = f(x, y)*3;
00077      \endcode
00078      *
00079      * The following, however, defines g as four dimensional:
00080      \code
00081      g(x, y, _) = f*3;
00082      \endcode
00083      *
00084      * It is equivalent to:
00085      *
00086      \code
00087      g(x, y, _0, _1) = f(_0, _1)*3;
00088      \endcode
00089      *
00090      * Expressions requiring differing numbers of implicit variables
00091      * can be combined. The left-hand-side of a definition injects
00092      * enough implicit variables to cover all of them:
00093      *
00094      \code
00095      Func h;
00096      h(x) = x*3;
00097      g(x) = h + (f + f(x)) * f(x, y);
00098      \endcode
00099      *
00100      * expands to:
00101      *
00102      \code
00103      Func h;
00104      h(x) = x*3;
00105      g(x, _0, _1) = h(_0) + (f(_0, _1) + f(x, _0)) * f(x, y);
00106      \endcode
00107      *
00108      * The first ten implicits, _0 through _9, are predeclared in this
00109      * header and can be used for scheduling. They should never be
00110      * used as arguments in a declaration or used in a call.
00111      *
00112      * While it is possible to use Var::implicit or the predeclared
00113      * implicits to create expressions that can be treated as small
00114      * anonymous functions (e.g. Func(_0 + _1)) this is considered
00115      * poor style. Instead use \ref lambda.
00116      */
