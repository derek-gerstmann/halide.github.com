15 /**
16  * This namespace defines a coding of the linearity of an expression as a simple integer, and
17  * a few functions for deciding linearity from a coded value. The code is fuzzy, in the sense that
18  * 0 is reserved for constant exprs, 1 for linear exprs, but any number >1 can be used to indicate
19  * an expr is non-linear. The functions included in this namespace should be used when deciding what
20  * case a particular int corresponds to, whereas the enum values can be used to set the value of a
21  * coded variable. We introduce this as in a namespace so that linearity can be stored and tracked
22  * in a simple integer. Users of the linearity test functions will therefore rarely need to directly
23  * use this namespace and do not need to be aware of another custom class.
24  */
