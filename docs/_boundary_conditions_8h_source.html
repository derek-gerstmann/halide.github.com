17 /** namespace to hold functions for imposing boundary conditions on
18  * Halide Funcs.
19  *
20  * All functions in this namespace transform a source Func to a
21  * result Func where the result produces the values of the source
22  * within a given region and a different set of values outside the
23  * given region. A region is an N dimensional box specified by
24  * mins and extents.
25  *
26  * Three areas are defined:
27  * The image is the entire set of values in the region.
28  * The edge is the set of pixels in the image but adjacent
29  * to coordinates that are not
30  * The interior is the image minus the edge (and is undefined
31  * if the extent of any region is 1 or less).
32  *
33  * If the source Func has more dimensions than are specified, the extra ones
34  * are unmodified. Additionally, passing an undefined (default constructed)
35  * 'Expr' for the min and extent of a dimension will keep that dimension
36  * unmodified.
37  *
38  * Numerous options for specifing the outside area are provided,
39  * including replacement with an expression, repeating the edge
40  * samples, mirroring over the edge, and repeating or mirroring the
41  * entire image.
42  *
43  * Using these functions to express your boundary conditions is highly
44  * recommended for correctness and performance. Some of these are hard
45  * to get right. The versions here are both understood by bounds
46  * inference, and also judiciously use the 'likely' intrinsic to minimize
47  * runtime overhead.
48  *
49  */

77 /** Impose a boundary condition such that a given expression is returned
78  * everywhere outside the boundary. Generally the expression will be a
79  * constant, though the code currently allows accessing the arguments
80  * of source.
81  *
82  * An ImageParam, Image<T>, or similar can be passed instead of a Func. If this
83  * is done and no bounds are given, the boundaries will be taken from the
84  * min and extent methods of the passed object.
85  *
86  * (This is similar to setting GL_TEXTURE_WRAP_* to GL_CLAMP_TO_BORDER
87  * and putting value in the border of the texture.)
88  */
89 // @{

91  const std::vector<std::pair<Expr, Expr> > &bounds);

93 template <typename T>
94 inline NO_INLINE Func constant_exterior(T func_like, Expr value) {
95  std::vector<std::pair<Expr, Expr> > object_bounds;
96  for (int i = 0; i < func_like.dimensions(); i++) {
97  object_bounds.push_back(std::make_pair(Expr(func_like.min(i)), Expr(func_like.extent(i))));
98  }
99 
100  return constant_exterior(Internal::func_like_to_func(func_like), value, object_bounds);
101 }
